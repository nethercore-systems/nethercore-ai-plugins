# Run-Based Design Reference

---

## Run Structure Templates

### Short Run (15-30 min)
```
[Start] → [Stage 1] → [Stage 2] → [Stage 3] → [Boss] → [End]
              ↓           ↓           ↓
          1-2 items   1-2 items   1-2 items
```
- 3-5 stages
- 5-8 items acquired
- 1-2 bosses
- Quick restart

### Medium Run (30-60 min)
```
[Start] → [Act 1: 3 stages] → [Mini-boss]
       → [Act 2: 3 stages] → [Mini-boss]
       → [Act 3: 2 stages] → [Final boss]
```
- 8-10 stages
- 15-20 items acquired
- 2-3 bosses
- Natural break points

### Long Run (60-120 min)
```
[Start] → [World 1: Hub + 3 zones] → [Boss]
       → [World 2: Hub + 4 zones] → [Boss]
       → [World 3: Hub + 3 zones] → [Final boss]
       → [Optional: Secret boss]
```
- 10-15+ stages
- 25-40 items acquired
- 3-5 bosses
- Save/resume supported

---

## Stage Progression

| Stage | Challenge | Rewards | Purpose |
|-------|-----------|---------|---------|
| 1 | Tutorial-level | Foundation items | Build start |
| 2-3 | Moderate | Core build items | Establish direction |
| 4-5 | Challenging | Synergy items | Power spike |
| 6-7 | Hard | Completion items | Test build |
| Final | Peak | Victory reward | Culmination |

---

## Restart Optimization

**Time from death to playing again:**
- Target: < 10 seconds
- Maximum: 30 seconds

**Reduce friction:**
- Instant respawn option
- Quick character select
- Skip cutscenes
- Remember last loadout

---

## Run Variety Checklist

- [ ] No two runs feel identical
- [ ] Different builds play differently
- [ ] Some rare events exist
- [ ] Player choices matter
- [ ] Skill still determines outcomes
