# Onboarding Checklist

---

## First Minute

- [ ] Immediate engagement (no long intros)
- [ ] Visual/audio hook
- [ ] Player has control within 30 seconds
- [ ] Simple first action

## First Five Minutes

- [ ] Core mechanic taught
- [ ] First success achieved
- [ ] Clear goal established
- [ ] First reward received
- [ ] Player feels competent

## First Session

- [ ] All core mechanics introduced
- [ ] Meaningful progress made
- [ ] Natural stopping point provided
- [ ] Reason to return established
- [ ] Difficulty feels appropriate

---

## Teaching Method Checklist

- [ ] Show before tell
- [ ] One mechanic at a time
- [ ] Safe environment for learning
- [ ] Immediate application required
- [ ] Failure is low-cost
- [ ] Success is clear and celebrated

---

## Onboarding Red Flags

| Problem | Fix |
|---------|-----|
| Wall of text | Replace with action |
| Too many mechanics at once | Sequence introduction |
| Forced slow sections | Add skip option |
| Repeated explanations | Make skippable |
| No early win | Add easy first challenge |
| Confusing UI | Simplify, introduce gradually |

---

## Tutorial Skip Considerations

- [ ] Can veteran players skip?
- [ ] Does skipping miss essential info?
- [ ] Is skip option visible?
- [ ] Can players return to tutorial?
